HTML

  • If an application is a movie, then HTML is the script. It is the blueprint that provides structure to the greater whole.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • the style element defines the CSS styles aattached to the website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to an HTML document. This is used for this website to add CSS and JavaScript to the site!
  • The meta element defines metadata such as character set, description, keywords, author and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • HTML attributes are always defined in the beginning element. They are NEVER defined in the closing element!

CSS

  • There are three ways to use CSS to style a webpage: inline CSS, internal CSS and external CSS.
  • External CSS is the most forgiving of the styles as it does not directly bake itself into the HTML file of an application.
  • The link function in HTML is used to apply the CSS style sheet to the website, making it simpler to code.
  • How to code a CSS rule: div(selector) {color (property): blue (value); text-align(property): justify(value);}
  • As stated above, a selector defines the element or attribute to which the rules or declarations will apply.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git is a free version control software that allows you to develop applications on your desktop.
  • GitHub is a cloud-based hosting service for your web applications.
  • Git Workflow: "git status" to verify location -> "cd" to change directory if needed ->
  • "git checkout -b feature/(insert-name)" To create a new branch -> "git add -A" to stage changes ->
  • "git commit -m 'add comment here' " to capture a snapshot of feature changes ->
  • "git push origin 'feature/branch-name'" to create a pull request. ->
  • Goto GitHub to confirm and merge pull request. Don't forget to add comments!
  • Use "git checkout main" to return to the main branch once you are finished with your feature branch.
  • Use "git pull origin main" to sync with cloud changes on the main branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Separation of concerns: refers to separating code into different files based on use;based on the concept that each section of code should have its own responsibility.
  • DRY (Do Not Repeat Yourself): a coding concept that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that neeeds to be written.
  • Array: a single variable that holds a group of data; This can store multiple items such as strings, numbers and booleans.
  • Function: a set of instructions that tells the computer how to perform a certain task.
  • To declare a function, use the "function" keyword followed by the variable you wish to define (ex. function MakePancakes() ).
  • Index: number associated with an item in an array, starts at 0.
  • "for" loop: Uses indices to perform tasks in an array, allowing a single line of code to be executed over and over.
  • The first statement in the loop indicates the starting point, defines a variable and its value (var x=0).
  • The second statement is the condition (as long as x is less than the length of the array, the loop will keep running).
  • The final statement allows the array to iterate over each item ("x++" is shorthand for x+1, which is used in this example).
  • Control flow dictates that JavaScript files are executed from top to bottom.
